# QA Lab SaaS Launch Plan (1-2 months)

## Current State Assessment

**Already Built:**
- Firebase auth (Google, email/password, guest)
- Workspace sync with real-time presence
- Team management with roles (QA Lead, Tester, Viewer)
- Jira integration (push to Jira)
- Activity audit trail
- Export to CSV/PDF
- Public report sharing
- Backup/restore
- Full QA loop: requirements, test cases, runs, bugs, plans, milestones

**Missing for SaaS:**
- No billing/subscription system
- No email notifications
- No onboarding wizard
- No public API
- No audit log export
- No custom fields
- No team invitation via email
- No session management / 2FA

## Phase 1: Billing & Monetization (Week 1-2)

### 1.1 Stripe Integration
- Add `stripe` and `@stripe/stripe-js` dependencies
- Create `src/utils/billing.js` — Stripe checkout, portal, subscription management
- Create `src/hooks/useSubscription.js` — subscription state hook
- Create `src/pages/BillingPage.jsx` — plan selection, payment history, manage subscription

### 1.2 Pricing Tiers
```
Free Tier:
- 1 project
- 25 test cases
- 10 team members
- 30-day history
- No API access

Pro ($12/user/month):
- Unlimited projects
- Unlimited test cases
- Unlimited team members
- 1-year history
- API access
- Slack integration
- Custom fields

Enterprise ($25/user/month):
- Everything in Pro
- Unlimited history
- SSO/SAML
- Audit log export
- Custom domain
- Priority support
- SLA guarantees
```

### 1.3 Usage Enforcement
- Add `src/utils/limits.js` — check limits before creating entities
- Add `src/components/UpgradeModal.jsx` — show when hitting limits
- Store subscription tier in Firestore `workspaces/{id}/subscription`

## Phase 4: Public API (Week 4-6)

### 4.1 API Design
```
Base URL: https://api.qalab.io/v1

Authentication:
- API key stored in Firestore
- Bearer token in Authorization header

Endpoints:
GET    /projects                    - List projects
GET    /projects/:id                - Get project
GET    /projects/:id/test-cases     - List test cases
POST   /projects/:id/test-cases     - Create test case
PUT    /projects/:id/test-cases/:id - Update test case
GET    /projects/:id/bugs           - List bugs
POST   /projects/:id/bugs           - Create bug
GET    /projects/:id/runs           - List runs
POST   /projects/:id/runs           - Create run
POST   /projects/:id/runs/:id/results - Submit run results
GET    /projects/:id/reports        - Get report data
```

### 4.2 API Key Management
- Create `src/pages/ApiKeysPage.jsx` — generate, revoke, view keys
- Store hashed keys in Firestore `workspaces/{id}/apiKeys`

### 4.3 API Documentation
- Create `src/pages/ApiDocsPage.jsx` — interactive API docs
- Or link to external docs (Swagger/OpenAPI)

### Files to Create/Modify:
- `src/pages/ApiKeysPage.jsx` (new)
